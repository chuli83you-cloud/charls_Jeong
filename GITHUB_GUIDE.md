# GitHub 초보자를 위한 가이드

## GitHub란?

GitHub는 코드 버전 관리 및 협업을 위한 웹 기반 플랫폼입니다.

### 주요 개념

1. **Repository(저장소)**
   - 프로젝트를 관리하는 공간
   - 파일과 폴더를 포함
   - 변경 이력이 기록됨

2. **Commit(커밋)**
   - 코드 변경사항을 저장하는 행위
   - 변경 내용에 대한 메시지를 포함
   - 프로젝트의 이력을 추적

3. **Branch(브랜치)**
   - 독립적인 작업 공간
   - "main" 브랜치가 기본값
   - 새로운 기능 개발에 사용

4. **Pull Request(PR)**
   - 변경사항을 검토하는 단계
   - 협업 시 필수
   - 코드 품질 관리

## 첫 단계

1. GitHub 계정 생성
2. Repository 생성
3. 파일 추가 및 커밋
4. 다른 사람과 협업 시작

## 유용한 팁

- README.md는 프로젝트 설명서
- .gitignore로 불필요한 파일 제외
- 의미 있는 커밋 메시지 작성
